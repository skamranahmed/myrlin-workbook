# Objective

I want to build my own project from scratch.

I have provided an existing open-source repository only as a reference. Do NOT treat it as the implementation that we should copy.

Your job in this phase is to understand the repository from a product and domain perspective.

Focus on answering these questions:

1. What problem is this project trying to solve?
2. Who are the intended users?
3. What workflows does it support?
4. What are the core concepts?
5. What assumptions does it make?
6. Which external tools or services does it integrate with?
7. Which features appear to be the primary value proposition?

Do NOT analyze implementation details unless they are necessary to understand the product.

If something is unclear, explicitly mention your assumptions.

Do not suggest improvements yet.

Treat this as a discovery exercise.

# Expected Output

repository-discovery.md

- Executive Summary

- Problem Statement

- Target Users

- Core Concepts

- Supported Workflows

- External Integrations

- Feature Summary

- Assumptions

- Questions / Unknowns
